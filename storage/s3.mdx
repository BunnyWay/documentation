Bunny Storage supports an S3-compatible API (currently in beta), allowing you to use familiar S3 tools, libraries, and workflows with your storage zones.

<Info>
  S3 compatibility is currently in beta. S3 compatibility must be enabled during
  storage zone creation and cannot be changed afterward.
</Info>

## Creating a storage zone with S3 compatibility

To create an S3-compatible storage zone:

1. Navigate to **Storage** → **Add Storage**
2. Enter a name (minimum 4 characters, letters, numbers, and dashes only)
3. Enable the **S3 Compatibility** option
4. Select your preferred storage tier
5. Choose your region (currently available: Frankfurt, New York, and Singapore)
6. Complete the setup process

<Note>
  S3 compatibility is currently only supported in Frankfurt, New York, and
  Singapore. Additional regions will be added later.
</Note>

## Supported S3 operations

Bunny Storage provides comprehensive S3 API compatibility for all your storage needs.

### Object operations

- **PUT Object**: Create and update objects
- **GET Object**: Read objects (range requests supported)
- **DELETE Object**: Delete objects (single and batch operations)
- **COPY Object**: Copy objects between locations
- **HEAD Object**: Get object metadata

### Directory listing

- **LIST Objects**: v1 and v2 API versions supported
- List with prefix filtering
- Paginated results
- Directory-style navigation
- Common prefixes support

### Multipart upload

- Initiate multipart upload
- Upload parts (up to 10,000 parts)
- Complete multipart upload
- Abort multipart upload
- List multipart uploads
- Recommended for files over 100MB

### S3 API compatibility matrix

| Operation          | Status           | Notes                        |
| ------------------ | ---------------- | ---------------------------- |
| PUT Object         | ✅ Supported     | Upload files                 |
| GET Object         | ✅ Supported     | Range requests supported     |
| DELETE Object      | ✅ Supported     | Single and batch operations  |
| LIST Objects       | ✅ Supported     | v1 and v2 API versions       |
| Multipart Upload   | ✅ Supported     | Recommended for files >100MB |
| COPY Object        | ✅ Supported     |                              |
| HEAD Object        | ✅ Supported     |                              |
| Pre-signed URLs    | ❌ Not supported | Temporary access links       |
| CORS Configuration | ❌ Not supported |                              |

## Benefits

- Use existing S3 tools and scripts
- Leverage popular S3 libraries across all programming languages
- Migrate from AWS S3 with minimal code changes
- Compatible with S3 GUI tools like Cyberduck, S3 Browser, and CloudBerry

## Impact on replication

Enabling S3 compatibility reduces the number of available replication points:

| Storage Tier | HTTP API/FTP     | S3-compatible |
| ------------ | ---------------- | ------------- |
| Standard     | Up to 9 regions  | 4 regions     |
| Edge (SSD)   | Up to 15 regions | 4 regions     |

<Warning>
  S3 compatibility cannot be changed after storage zone creation. If you need
  more than 4 replication points, use the HTTP API or FTP instead.
</Warning>

## S3 credentials

After creating your S3-compatible storage zone, you'll receive:

- **Access Key ID**: Your unique identifier
- **Secret Access Key**: Your authentication credential
- **Endpoint URL**: Your storage zone's S3-compatible endpoint (format: `https://[region]-s3.storage.bunnycdn.com`)
- **Region**: The primary region for your storage zone (e.g., `de` for Frankfurt, `ny` for New York, `sg` for Singapore)

Find your credentials in the **FTP & API Access** tab of your storage zone.

When using S3-compatible tools, configure them with:

- **Endpoint**: `https://[region]-s3.storage.bunnycdn.com`
- **Bucket name**: Your storage zone name
- **Access Key ID**: From your storage zone settings
- **Secret Access Key**: Your storage zone password

## Usage

### AWS CLI

<Note>
  You must use `--endpoint-url https://[region]-s3.storage.bunnycdn.com` with
  each command or set up an [AWS CLI
  profile](https://docs.aws.amazon.com/cli/v1/userguide/cli-configure-files.html#cli-configure-files-methods)
  with the endpoint configured.
</Note>

```bash
# Configure AWS CLI with your storage zone credentials
aws configure --profile bunny
# AWS Access Key ID: your-access-key-id
# AWS Secret Access Key: your-storage-password
# Default region name: your-region (e.g., de, ny, sg)
# Default output format: json

# Upload a file
aws s3 cp /path/to/local/file s3://bucket-name/path/to/destination/ \
  --profile bunny \
  --endpoint-url https://[region]-s3.storage.bunnycdn.com

# List objects
aws s3 ls s3://bucket-name/ --recursive \
  --profile bunny \
  --endpoint-url https://[region]-s3.storage.bunnycdn.com

# Create a directory
aws s3api put-object --bucket bucket-name --key folder-name/ \
  --profile bunny \
  --endpoint-url https://[region]-s3.storage.bunnycdn.com

# Download a file
aws s3 cp s3://bucket-name/path/to/file /path/to/local/destination/ \
  --profile bunny \
  --endpoint-url https://[region]-s3.storage.bunnycdn.com
```

### rclone

Configure rclone for Bunny Storage by running `rclone config` and following these steps:

1. Choose `n` for a new remote
2. Name it (e.g., `bunnycdn`)
3. Choose `s3` as the storage type
4. For the S3 provider, choose `Other` (since BunnyCDN isn't officially listed)
5. Press `1` to select `Enter AWS credentials in the next step`
6. Enter the `access_key_id` and `secret_access_key` for your S3-enabled storage zone
7. Leave the region and other fields as default unless necessary
8. Enter the custom endpoint URL: `https://[region]-s3.storage.bunnycdn.com` (e.g., `https://de-s3.storage.bunnycdn.com`)
9. Leave the rest as the defaults

Alternatively, configure rclone via command line:

```bash
# Configure rclone
rclone config create bunnycdn s3 \
  provider=Other \
  access_key_id=your-access-key-id \
  secret_access_key=your-storage-password \
  endpoint=https://[region]-s3.storage.bunnycdn.com

# Upload a file
rclone copy /path/to/local/file bunnycdn:bucket-name/path/to/destination/

# Download a file
rclone copy bunnycdn:bucket-name/path/to/file /path/to/local/destination/

# List files
rclone ls bunnycdn:bucket-name/

# Sync a directory
rclone sync ./local-dir bunnycdn:bucket-name/remote-dir/
```
