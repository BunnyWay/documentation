## Width

### Width vs height

If both **width** and **height** parameters are defined, the one that produces smaller image while preserving aspect ratio will take effect.

Lets look at the following image. Its dimensions are 500x333.

<Frame caption="Starting image.">
  <img src="/images/docs/bunny_in_grass.jpg" />
</Frame>

<Frame caption="?width=&height=">
  <img src="/images/docs/bunny_in_grass.jpg" />

**300\*\***250\*\*

</Frame>

Now we want to apply the following query parameters: _?width=300\&height=250_ to it. Lets see what results we get if we apply them separately.

<Frame caption="?width=">
  <img src="/images/docs/bunny_in_grass.jpg" />

**300**

</Frame>

<Frame caption="height=">
  <img src="/images/docs/bunny_in_grass.jpg" />

**_250_**

</Frame>

The second image is smaller and that is also the result of _?=width=300\&height=250_.

### Priority

[Crop](/docs/cropping) operations have priority over the _width_ or _height_ commands. If any crop is defined in the query it will be applied first.
