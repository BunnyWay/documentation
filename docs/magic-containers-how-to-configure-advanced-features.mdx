In this guide, we will walk you through the process of customizing your containers' behavior to align with your specific use cases and operational requirements. bunny.net offers a versatile platform that empowers you to fine-tune your containers, providing you with the tools you need to optimize their performance and functionality.

# What you'll need

Before you dive in, make sure you have the following prerequisites in place:

* A [bunny.net](https://bunny.net/) account ([Log in](https://dash.bunny.net/auth/login?pk_buttonlocation=menu) or sign up for a [free trial](https://dash.bunny.net/auth/register)).
* Ensure that you have already deployed the application for which you want to configure health checks. Health checks are only relevant when you have an application up and running within a container, and these checks help ensure the application's reliability and availability.

# Configuring advanced features

To configure advanced features for your containers on bunny.net, please follow the steps below:

1. Login to [bunny.net dashboard](https://dash.bunny.net/auth/login?pk_buttonlocation=menu).

2. Click Magic Containers and locate the container for which you want to configure advanced features.

3. Click **Container Settings** and select the **Advanced** tab.

![](/images/docs/8e2a2cdaef5760ac80ed1d5cf7bb261992e8748211229bb8d64119714b55b3df-image.png)

Advanced features are designed to offer enhanced control over your containers. Below are the types and descriptions of the advanced features you can configure:

* **Always pull the image**: When enabled, this feature ensures that the application pulls the container image from the registry whenever a new node is started. This helps guarantee that your container is always up-to-date.
* **Startup command**: This feature allows you to specify a custom command that will execute when the container is launched. This provides control over the initial behavior of your container.
* **Container Arguments**: This feature enables you to configure the arguments that will be added to the container's entry point when starting the image. This is useful for passing runtime parameters to your container.
* **Working Dir**: This feature allows you to set the working directory of the container runtime. Configuring the working directory is important for applications that rely on specific file paths.
