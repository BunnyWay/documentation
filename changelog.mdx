<Update label="2026-03-18" tags={["New", "Stream"]}>
  ## Vimeo2Bunny migration tool

  A new CLI tool makes it easy to migrate your videos from Vimeo to Bunny Stream. Videos transfer directly via URL fetch, so nothing is downloaded to your machine. [Learn more](/stream/vimeo2bunny).
</Update>

<Update label="2026-03-18" tags={["Improvement", "CDN"]}>
  ## Edge Rules pattern matching

  Edge Rules now support Lua-based pattern matching for more flexibility when matching structured request values such as URLs, headers, cookies, and query strings. [Learn more](/cdn/edge-rules/pattern-matching).
</Update>

<Update label="2026-03-17" tags={["New", "Database"]}>
  ## Database Shell

  A new interactive SQL shell (`bsql`) lets you query and manage your Bunny Database directly from the terminal with dot-commands, multiple output formats, and persistent history. [Learn more](/database/connect/database-shell).
</Update>

<Update label="2026-03-16" tags={["New", "Stream"]}>
  ## Webhook signature validation

  You can now verify the authenticity of Stream webhook callbacks using signature validation to ensure requests are genuinely from bunny.net. [Learn more](/stream/webhooks).
</Update>

<Update label="2026-03-12" tags={["New", "Stream"]}>
  ## Stream Player 2.0

  The new Bunny Stream video player is here with a modern interface and improved performance, enabled by default for all new video libraries. [Learn more](/stream/player).
</Update>

<Update label="2026-03-10" tags={["New", "Magic Containers"]}>
  ## Quick Deploy

  A new streamlined deployment flow lets you go from zero to deployed in seconds with everything in a single form. [Learn more](/magic-containers/quick-deploy).
</Update>

<Update label="2026-03-10" tags={["New", "Magic Containers"]}>
  ## Templates

  Deploy pre-built application templates with just a few clicks, including ready-to-run container images and optional sidecar databases. [Learn more](/magic-containers/templates).
</Update>

<Update label="2026-03-09" tags={["New", "CDN"]}>
  ## JA4 fingerprinting

  Identify clients using TLS fingerprinting with JA4. bunny.net automatically computes a JA4 fingerprint for incoming HTTPS requests and exposes it via the `CDN-JA4` request header. [Learn more](/cdn/security/ja4-fingerprinting).
</Update>

<Update label="2026-03-09" tags={["Improvement", "CDN"]}>
  ## CDN logging enhancements

  The logging overview and permanent log storage documentation have been updated with clearer guidance on log retention, storage configuration, and forwarding options. [Learn more](/cdn/logging).
</Update>

<Update label="2026-03-05" tags={["Improvement", "Stream"]}>
  ## Player migration guide

  A new migration guide helps you transition custom markup from the legacy player to the new Bunny Stream Player 2.0. [Learn more](/stream/custom-head-html-migration-guide).
</Update>

<Update label="2026-02-19" tags={["New", "Magic Containers"]}>
  ## Database tutorials for Magic Containers

  New step-by-step tutorials for running PostgreSQL, MariaDB, Redis, and ClickHouse on Magic Containers with persistent volumes. [Learn more](/magic-containers/guides/postgresql).
</Update>

<Update label="2026-02-13" tags={["Improvement", "Storage"]}>

  New regional server endpoints have been added for London, Stockholm, Sao Paulo, and Johannesburg. [Learn more](/storage/http).
</Update>

<Update label="2026-02-11" tags={["New", "Stream"]}>
  ## Mobile SDK token authentication

  You can now use token authentication with the Bunny Stream Mobile SDKs to securely control video playback access in your mobile applications. [Learn more](/stream/mobile-sdk-token-authentication).
</Update>

<Update label="2026-02-09" tags={["New", "Magic Containers"]}>
  ## Graceful shutdown

  Magic Containers now supports graceful shutdown with configurable grace periods, giving your application time to clean up resources and complete in-flight requests. [Learn more](/magic-containers/graceful-shutdown).
</Update>

<Update label="2026-01-30" tags={["New", "Magic Containers"]}>

<Update label="2026-01-29" tags={["New", "DNS"]}>
  ## DNS documentation launch
